変数の命名規則

仕様としての命名規則

  • 使用できる文字は、英文字（a-z, A-Z）、数字（0-9）、アンダースコア（_）、ドル記号（$）のいずれかである
  • １文字目に数字を使用することはできない
  • 変数名に予約語を使用することはできない
      // 使用できる変数名
      key  sum1  _name  $width  sum_all  sumAll  newValue

      // 使用できない変数名
      1st（最初の文字が数字） sum-all（ハイフンは使えない） new（予約語）

慣習としての命名規則

  • キャメルケースで変数の意味が理解できる名前をつける
  • 基本的に英単語を使う
  • ループカウンタ変数には、i、j、k などを使う
  • 定数は大文字にする（例：MAX_SIZE）
  • 真偽値を表す変数は、先頭に is や has を使う（例：isMouseDown, hasError）
  • コンストラクタの名前には、アッパーキャメル記法を使う

関係演算子

演算子 意味 例 例の値
== 等価 a == b a と b の値が等しいとみなせるときは true、それ以外は false
!= 等価でない a != b a と b の値が異なるとみなせるときは true、それ以外は false
=== 同値 a === b a と b の値が同一である（型と値が等しい）ときは true、それ以外は false
!=== 同値でない a !== b a と b の値が同一でないときは true、それ以外は false
< より小さい a < b a の値が b の値より小さいときは true、それ以外は false
> より大きい a > b a の値が b の値より大きいときは true、それ以外は false
<＝ 以下 a <＝ b a の値が b の値以下のときは true、それ以外は false
>＝ 以上 a >＝ b a の値が b の値以上のときは true、それ以外は false

抽象比較と厳密比較

      // 抽象比較
      1 == 1 // true
      1 == '1' // true
      1 == true // true
      '' == false // true
      '0' == false // true

      // 厳密比較
      1 === 1 // true
      1 === '1' // false
      1 === true // false
      '' === false // false
      '0' === false // false

プログラムの記述方法

HTML 内に記述

  • HTML ファイル内のどこかの script タグ内に書く。
  • script タグは、body の閉じタグ直前が推奨。
      <head>
        ~
        <script>
          // 処理を書く
        </script>
        ~
      </head>
      <body>
        <script>
          // 処理を書く
        </script>
      </body>

外部ファイルに記述する

  • 外部ファイルの拡張子は「.js」にする。
  • HTML ファイル内のどこかの script タグの src 属性にファイルパス
    または、URL を書く。
  • script タグは、body の閉じタグ直前が推奨。
      <head>
        <script src="[js ファイルのパス]"></script>
        <!-- または -->
        <script src="[js ファイルの URL]"></script>
      </head>
      <body>
        <script src="[js ファイルのパス]"></script>
        <!-- または -->
        <script src="[js ファイルの URL]"></script>
      </body>

HTML 属性に記述する

  • HTML タグのイベント属性に記述する。
  • ソースコードの可読性が悪くなるため、非推奨。
      // 属性
      on[イベント名]="[js の処理]"

      // 例
      <button onclick="alert('クリック！');">ボタン</button>
